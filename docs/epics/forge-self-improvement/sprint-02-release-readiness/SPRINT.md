# Sprint 02 — Release Readiness

A documentation pass to make the repo understandable enough to share. One docs ticket (T02): bring the
README in line with what Forge Core actually does today — accurate status, the full command surface,
install/setup, the v1 safety model, known limitations, and a quickstart. No source changes.

**Done means:** the README no longer claims the importer/execution are unbuilt; every shipped command and
wrapper is documented; install/setup and the `FORGE_BIN`/`FORGE_REPO` resolution are explained; the
conservative, human-gated v1 safety model and its limitations are stated honestly (no maturity or autonomy
overclaims); a quickstart runs against an existing epic.

**Halt-triggers:** any change outside `README.md`; any source/config/contract edit; any claim of full
autonomy or unsupervised readiness; a failing verify command.
